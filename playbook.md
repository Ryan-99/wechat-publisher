# Evernow 写作 Playbook
# 优先级：confidence ≥ 5 的规则在写作时作为硬性约束执行
# confidence < 5 的规则作为软性参考
# Playbook 是 WeWrite 中优先级最高的配置层，覆盖 persona 和 writing-guide

---

## 规则 1: 事实性核查（confidence: 5）

**描述**：文章中所有涉及产品定价、免费/付费声明、产品名称、功能描述、数据引用的语句，在写作完成后必须经过 WebSearch 交叉验证。

**执行时机**：Step 4 写作完成后、Step 5 验证之前（在快速自检之后、SEO 验证之前插入）

**核查清单**：

- [ ] 产品名称是否为当前最新名称（如 Moltbot → OpenClaw)
- [ ] 定价/免费声明是否准确(如"免费"→是否真的免费？试用和正式免费的区别)
- [ ] 具体数据/百分比是否可追溯来源(如"效率提升60%"来自哪份报告？)
- [ ] 产品功能描述是否与当前版本一致
- [ ] GitHub Star 数是否为近似当前值(允许误差 ±20%)

**修复标准**：
- 命中 → 立即修正，使用 WebSearch 验证后的准确信息
- 无法验证 → 删除或改为模糊表述("据公开数据"而非"据 XX 报告")
- 每次修正记录在 history.yaml 的 notes 字段中

---

## 规则 2: 安全视角渗透（confidence: 4）

**描述**：Evernow 定位是"安全工程师的 AI 实验室"，涉及安全相关话题时优先从安全从业者角度切入。

**执行方式**：
- 涉及 AI 工具时，关注安全/隐私面(数据泄露、API 调用链、本地 vs 云端)
- 涉及自动化时，关注权限边界和风险控制
- 不强行加安全内容，但在涉及工具选择和评估时自然带入安全视角

---

## 规则 3: 工具推荐诚实性（confidence: 5）

**描述**：推荐工具时必须区分"我用过"和"听说不错"。

**执行方式**：
- 真实使用过的工具 → 可以写具体体验
- 仅了解但未深度使用的工具 → 标注为"值得关注"或"据说不错"，不要伪装成亲测
- 没有使用过的工具 → 不要推荐，可以列入"如果你感兴趣可以看看"的弱推荐

---

## 规则 4: 文章输出目录（confidence: 5）

**描述**：生成的文章及其所有相关文件（markdown、HTML、图片）统一输出到当前工作目录的 projects 文件夹下，不放在 skill 安装目录里。

**执行方式**：

1. Step 4 写作完成后，在 `~/Documents/code/claude/projects/` 下创建以文章主题命名的目录（如 `ai-agent-automation-2026/`）
2. 将 markdown、HTML、封面图、配图全部放入该目录
3. history.yaml 中的 output_file 字段指向该目录下的 markdown 文件
4. 同时在 skill 的 output/ 目录保留一份 markdown 副本（供 WeWrite 内部流程使用）
